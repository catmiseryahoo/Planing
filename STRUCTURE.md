# Структура приложения для визуального планирования проектов

## 1. Идея продукта

Приложение помогает видеть процесс проекта целиком: этапы, задачи, ответственных, сроки и текущий статус.

Главная ценность:

> Один экран показывает, что происходит в проекте, кто за что отвечает и какие задачи требуют внимания.

Продукт не должен быть похож на сложную BPM-систему с регламентами и тяжелой аналитикой. Основной акцент - на визуальной карте проекта и понятной работе команды.

## 2. Что исключаем

- Регламенты.
- Сложную аналитику.
- BPMN как обязательный стандарт.
- Аудит процессов.
- Документы SOP.
- Тяжелую процессную методологию.

## 3. Что оставляем

- Визуальную карту проекта.
- Этапы проекта.
- Задачи внутри этапов.
- Ответственных за задачи.
- Сроки и дедлайны.
- Статусы задач.
- Комментарии.
- Уведомления.
- Шаблоны типовых проектов.

## 4. Основной сценарий пользователя

1. Пользователь создает проект.
2. Разбивает проект на этапы.
3. Добавляет задачи в каждый этап.
4. Назначает ответственных.
5. Указывает сроки.
6. Связывает этапы и задачи на визуальной карте.
7. Следит за статусами и просрочками.
8. Открывает карточку задачи, чтобы уточнить детали или оставить комментарий.

## 5. Главный экран приложения

Главный экран - карта проекта.

Структура экрана:

```text
┌─────────────────────────────────────────────────────────────┐
│ Проект: Запуск сайта        [Карта] [Доска] [Список] [+]    │
├───────────────┬─────────────────────────────────────────────┤
│ Проекты       │                                             │
│               │   Брифинг -> Дизайн -> Разработка -> Запуск │
│ Запуск сайта  │      │          │             │          │  │
│ CRM           │    задачи     задачи        задачи     задачи│
│ Реклама       │                                             │
│               │   Цветом видно статус, срок и ответственного │
├───────────────┴─────────────────────────────────────────────┤
│ Просрочено: 3    В работе: 12    Готово: 24                 │
└─────────────────────────────────────────────────────────────┘
```

Слева расположен список проектов. В центре - большая визуальная карта. Справа при выборе этапа или задачи открывается панель свойств.

## 6. Основные разделы приложения

### 6.1. Проекты

Экран со списком всех проектов.

На карточке проекта:

- название проекта;
- текущий статус;
- ближайший дедлайн;
- количество задач;
- количество просроченных задач;
- ответственный за проект.

### 6.2. Карта проекта

Главный рабочий экран.

На карте отображаются:

- этапы проекта;
- задачи внутри этапов;
- связи между этапами;
- ответственные;
- сроки;
- статусы;
- индикаторы просрочки.

### 6.3. Мои задачи

Личный экран исполнителя.

Группы задач:

- сегодня;
- на этой неделе;
- просрочено;
- на проверке;
- завершено.

### 6.4. Команда

Экран участников проекта.

Для каждого участника:

- имя;
- роль;
- активные задачи;
- ближайший дедлайн;
- простая оценка нагрузки: свободен, нормально, перегружен.

### 6.5. Календарь

Календарный вид сроков.

Показывает:

- дедлайны задач;
- старт и окончание этапов;
- задачи по дням и неделям;
- просроченные задачи.

### 6.6. Шаблоны

Галерея типовых проектов.

Примеры шаблонов:

- запуск сайта;
- рекламная кампания;
- разработка продукта;
- внедрение CRM;
- организация мероприятия;
- найм сотрудника.

## 7. Виды проекта

В проекте нужен переключатель представлений:

```text
[Карта] [Доска] [Список] [Календарь]
```

### Карта

Основной режим. Показывает проект как визуальный процесс из этапов, задач и связей.

### Доска

Задачи сгруппированы по статусам:

- запланировано;
- в работе;
- на проверке;
- готово;
- просрочено.

### Список

Табличный режим для быстрого редактирования:

- задача;
- этап;
- ответственный;
- срок;
- статус;
- приоритет.

### Календарь

Режим для просмотра дедлайнов и планирования сроков.

## 8. Карточка этапа

Карточка этапа на карте должна быть компактной и визуальной.

Пример:

```text
┌─────────────────────┐
│ Дизайн              │
│ 4 задачи · 2 в работе│
│                     │
│ Мария  до 17 мая    │
│ Илья   до 20 мая    │
│ 1 просрочена        │
└─────────────────────┘
```

Поля этапа:

- название;
- описание;
- список задач;
- ответственный за этап;
- срок начала;
- срок окончания;
- статус;
- связанные этапы.

## 9. Карточка задачи

Поля задачи:

- название;
- описание;
- проект;
- этап;
- ответственный;
- срок;
- статус;
- приоритет;
- чек-лист;
- комментарии;
- дата создания;
- дата завершения.

Пример:

```text
Прототип главной страницы

Статус: В работе
Ответственный: Мария
Срок: 17 мая
Этап: Дизайн
Приоритет: Средний

Описание:
Собрать первый прототип главной страницы.

Чек-лист:
[ ] Первый экран
[ ] Блок преимуществ
[ ] Форма заявки
```

## 10. Статусы

Базовые статусы:

- запланировано;
- в работе;
- на проверке;
- готово;
- просрочено.

Цветовая логика:

- серый - запланировано;
- синий - в работе;
- желтый - на проверке;
- зеленый - готово;
- красный - просрочено.

## 11. Основные сущности

### Проект

- id;
- название;
- описание;
- владелец;
- дата начала;
- дата окончания;
- статус;
- участники.

### Этап

- id;
- project_id;
- название;
- описание;
- порядок;
- дата начала;
- дата окончания;
- статус.

### Задача

- id;
- project_id;
- stage_id;
- название;
- описание;
- ответственный;
- срок;
- статус;
- приоритет.

### Пользователь

- id;
- имя;
- email;
- роль;
- аватар.

### Комментарий

- id;
- task_id;
- author_id;
- текст;
- дата создания.

## 12. MVP

Минимальная первая версия:

1. Создание проекта.
2. Создание этапов проекта.
3. Визуальная карта этапов.
4. Создание задач внутри этапов.
5. Назначение ответственных.
6. Установка сроков.
7. Статусы задач.
8. Экран "Мои задачи".
9. Простые шаблоны проектов.

## 13. Дизайн-направление

Интерфейс должен быть легким, визуальным и рабочим.

Рекомендации:

- светлый фон;
- чистая карта проекта в центре;
- компактные карточки задач;
- цветовая кодировка статусов;
- минимум декоративных элементов;
- быстрые действия прямо на карте;
- боковая панель для деталей;
- понятные иконки для статусов, сроков и ответственных.

Приложение должно ощущаться проще, чем Jira, структурнее, чем Miro, и нагляднее, чем обычный список задач.
