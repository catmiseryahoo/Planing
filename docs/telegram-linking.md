# Привязка личного Telegram к Orbita

## Для пользователя

1. Откройте Orbita.
2. Перейдите в личный кабинет.
3. В блоке Telegram нажмите `Получить код`.
4. Откройте корпоративного Telegram-бота.
5. Отправьте боту команду из личного кабинета в формате `/start 123456`.
6. Дождитесь ответа бота: `Telegram привязан к Orbita`.
7. В личном кабинете включите канал `Telegram`, если руководитель разрешил отправку уведомлений.

Код действует 15 минут. Если срок истёк, получите новый код в личном кабинете.

## Для администратора

### Что именно нужно настроить

Для Telegram-уведомлений в Orbita всегда нужен бот. Он принимает коды привязки от сотрудников и отправляет личные уведомления.

Telegram-канал не обязателен. Он нужен только если организация хочет получать копию уведомлений в общий корпоративный канал, например для проектного офиса, службы контроля или руководства. Если нужны только личные уведомления сотрудникам, канал можно не создавать.

### Вариант 1. Только личные уведомления сотрудникам

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Задайте название бота, например `Orbita Planner`.
4. Задайте username бота. Он должен заканчиваться на `bot`, например `OrbitePlaner_bot`.
5. Сохраните токен, который выдаст `@BotFather`.
6. Добавьте токен и secret webhook в Supabase:

```bash
npx supabase secrets set TELEGRAM_BOT_TOKEN="токен_бота"
npx supabase secrets set TELEGRAM_WEBHOOK_SECRET="случайная_длинная_строка"
```

7. Разверните функции:

```bash
npx supabase functions deploy create-telegram-link-code
npx supabase functions deploy telegram-webhook --no-verify-jwt
npx supabase functions deploy configure-telegram-webhook --no-verify-jwt
npx supabase functions deploy dispatch-message-notifications
```

8. Подключите webhook бота к Supabase Function, не раскрывая токен бота локально:

```bash
curl -X POST "https://wqfpksyemvaxncsqwuzm.supabase.co/functions/v1/configure-telegram-webhook" \
  -H "x-setup-secret: $TELEGRAM_WEBHOOK_SECRET"
```

9. В Orbita откройте `Админ-панель` -> `Каналы организации`.
10. Включите `Telegram`.
11. В поле `Название бота или канала` укажите username бота, например `@OrbitePlaner_bot`.
12. Поле `@channel или chat_id` можно оставить пустым, если общий канал не нужен.
13. Нажмите `Сохранить`.
14. Попросите сотрудников привязать личный Telegram по инструкции выше.

После этого личные Telegram-уведомления будут уходить сотрудникам, у которых:

- Telegram привязан через код в личном кабинете;
- в карточке сотрудника включён канал `Telegram`;
- сообщение адресовано этому сотруднику или его проектному чату.

### Вариант 2. Личные уведомления плюс общий канал организации

Сначала выполните шаги из варианта 1. Затем настройте канал:

1. Создайте Telegram-канал.
2. Задайте каналу публичный username, например `@OrbitePlaner`.
3. Добавьте созданного бота в администраторы канала.
4. Дайте боту право публиковать сообщения.
5. В Orbita откройте `Админ-панель` -> `Каналы организации`.
6. Включите `Telegram`.
7. В поле `Название бота или канала` укажите username бота, например `@OrbitePlaner_bot`.
8. В поле `@channel или chat_id` укажите username канала, например `@OrbitePlaner`.
9. Нажмите `Сохранить`.
10. Отправьте тестовое сообщение в мессенджере Orbita.

Если канал приватный, вместо `@channel` нужен `chat_id` канала. Его можно получить через служебные инструменты Telegram Bot API после добавления бота в канал, но для первой настройки проще использовать публичный username канала.

### Проверка

1. Сотрудник открывает личный кабинет, нажимает `Получить код`.
2. Сотрудник переходит по ссылке на корпоративного бота.
3. Сотрудник отправляет боту `/start 123456`.
4. Бот отвечает: `Telegram привязан к Orbita`.
5. В карточке сотрудника включён канал `Telegram`.
6. В мессенджере Orbita отправляется новое сообщение этому сотруднику.
7. Сотрудник получает личное уведомление от бота.
8. Если настроен общий канал, копия уведомления появляется в канале.

### Частые причины, если уведомления не приходят

- Сотрудник не отправил боту `/start код`.
- Код истёк: он действует 15 минут.
- В карточке сотрудника выключен канал `Telegram`.
- В настройках организации выключен Telegram.
- В поле `Название бота или канала` указан канал, а не бот.
- В поле `@channel или chat_id` указан бот, а не канал.
- Бот не является администратором канала.
- У бота нет права публиковать сообщения в канале.
- Канал приватный, но указан `@username`, которого у приватного канала нет.

Важно: бот не может первым написать человеку. Пользователь должен сам открыть бота и отправить `/start код`.
